## নিয়মাবলী

- **সরাসরি `main` ব্রাঞ্চে push করা যাবে না।** `main` এ যেকোনো পরিবর্তন GitHub-এ Pull Request এর মাধ্যমে করতে হবে।
- সবসময় **conventional commit message** ব্যবহার করতে হবে।
- প্রতিটি commit হবে **atomic** — একটি commit এ একটি মাত্র লজিক্যাল পরিবর্তন।
- সঠিক **branch naming convention** অনুসরণ করতে হবে: উপযুক্ত prefix এবং বর্ণনামূলক নাম দিতে হবে।
- ফাইল **আলাদা আলাদা ভাবে stage** করতে হবে। `git add .` ব্যবহার করা যাবে না।

## Task 0 — Command Log (বাধ্যতামূলক, চলমান)

git এর কাজ শুরু করার আগে প্রজেক্টের রুটে `COMMANDS.md` নামে একটি ফাইল তৈরি করো।

পুরো পরীক্ষার সময় তুমি যেসব **git command** চালাবে সেগুলো ক্রমানুসারে এই ফাইলে লিখে রাখবে — প্রতি লাইনে একটি command, চাইলে ছোট মন্তব্য দিয়ে কেন চালিয়েছো তা বোঝাতে পারো।

**COMMANDS.md এর নিয়ম:**

- এই ফাইলটি **কখনোই commit করা যাবে না**। এটি সবসময় untracked অবস্থায় থাকবে।
- পরীক্ষা শেষে এই ফাইলটি Slack-এ পরীক্ষকের কাছে জমা দিতে হবে।

*এই task বাধ্যতামূলক কিন্তু সরাসরি নম্বর নেই। তবে যদি `COMMANDS.md` commit হয়ে যায় বা অনুপস্থিত থাকে, "Selective staging" থেকে নম্বর কাটা যাবে।*

## Task 2 — Feature: Header Title ও Background Color পরিবর্তন

১. `main` থেকে একটি উপযুক্ত নামের branch তৈরি করো এবং সেই branch এ যাও।

> এই দুটি অবশ্যই **আলাদা দুটি commit** হতে হবে — একটি HTML পরিবর্তনের জন্য, একটি CSS পরিবর্তনের জন্য।

এই branch এর জন্য এখনই PR খুলো না। Task 4 এ আবার এখানে ফিরে আসবে।

## Task 3 — Feature: নতুন Operation যোগ করা (modulo)

১. `main` এ ফিরে যাও।
২. `main` থেকে একটি উপযুক্ত নামের branch তৈরি করো এবং সেই branch এ যাও।
৩. `app.js` এর `operations` object এ `modulo: (a, b) => a % b` যোগ করো।
৪. সঠিক conventional commit message দিয়ে commit করো।
৫. `index.html` এ `#operation` `<select>` এর ভেতরে নতুন `<option value="modulo">modulo</option>` যোগ করো।
৬. এটি **আলাদাভাবে** সঠিক conventional message দিয়ে commit করো।
৭. branch টি remote এ push করো।
৮. GitHub এ `main` এ Pull Request খুলো।
৯. PR টি merge করো — **merge commit** তৈরি হতে হবে (squash বা rebase merge নয়)।

> `app.js` এবং `index.html` এর পরিবর্তন **আলাদা দুটি commit** হতে হবে।

## Task 4 — Rebase Workflow

১. Task 2 এ তৈরি করা branch এ যাও।
২. আপডেটেড `main` (যাতে Task 3 এর modulo merge অন্তর্ভুক্ত) এর উপর rebase করো। Rebase শেষে তোমার Task 2 এর commit গুলো সর্বশেষ `main` এর উপরে বসবে।
৩. branch টি remote এ push করো (history পরিবর্তন হওয়ার কারণে `--force-with-lease` গ্রহণযোগ্য)।
৪. GitHub এ `main` এ Pull Request খুলে merge করো।

## Task 5 — Merge Conflict Resolve

repo তে আগে থেকেই `feature/update-header-test` নামে একটি branch আছে। এই branch ও Task 2 এ তোমার পরিবর্তিত nav title এবং navbar color পরিবর্তন করে — ফলে overlap হবে।

এই branch থেকে সরাসরি `main` এ PR খুলতে গেলে GitHub বলবে যে **automatic merge সম্ভব নয়**। প্রথমে local এ conflict resolve করতে হবে।

১. Local এ `feature/update-header-test` branch checkout করো।
২. এই branch এ `main` merge করো।
৩. Merge conflict আসবে — এটা প্রত্যাশিত। Resolve করো:
  - nav title এর জন্য: **তোমার** Task 2 এর version রাখো (`Rainbow IT - Smart Calculator`)।
  - navbar color এর জন্য: `feature/update-header-test` এর version রাখো (`#2c3e50`)।
৪. Resolve করা ফাইল গুলো stage করে সঠিক message দিয়ে merge commit করো।
৫. আপডেট হওয়া branch টি remote এ push করো।
৬. GitHub এ `main` এ Pull Request খুলে merge করো।

১. `main` এ যাও এবং সর্বশেষ পরিবর্তন pull করো।
২. `main` থেকে একটি উপযুক্ত নামের bugfix branch তৈরি করো।
৩. `app.js` এ bug টি খুঁজে বের করো। `subtract` function এখন দুটি সংখ্যা বিয়োগের বদলে যোগ করছে। এটি ঠিক করো।
৪. সঠিক conventional commit message দিয়ে তোমার সমাধান commit করো।
৫. branch টি remote এ push করো।
৬. GitHub এ `main` এ Pull Request খুলে merge করো।
